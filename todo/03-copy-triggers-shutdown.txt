ISSUE: copy_processed() sends a shutdown event -- hidden side effect

In ui.rs:457, copy_processed() sends UiShutdown after copying text. This means "copy" is
actually "copy and exit." The function name doesn't convey this, and anyone reading the
Message::Copy handler would need to trace into copy_processed() to discover the app terminates.
This is a business-logic coupling that should be explicit in the message handler.

SOLUTION:

Move the shutdown side effect out of copy_processed() and into the Message::Copy handler in
the update() function. Make copy_processed() do only what its name says -- return the processed
text for clipboard copying. Then in the Message::Copy match arm, after the clipboard write,
explicitly send the UiShutdown event. This makes the "copy then exit" behavior visible at the
call site:

    Message::Copy => {
        if let Some(text) = state.ui.copy_processed() {
            state.ui.send_event(AppEventKind::UiShutdown);
            return iced::clipboard::write::<Message>(text);
        }
        Task::none()
    }
