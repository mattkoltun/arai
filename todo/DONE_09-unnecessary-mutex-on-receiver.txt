ISSUE: app_event_rx receiver is wrapped in Mutex but only accessed from one thread

controller.rs:19: app_event_rx: Mutex<AppEventReceiver> -- this receiver is only read in
Controller::run(), which runs on exactly one thread. The Mutex exists solely to make the type
Sync (required by Arc<Controller>), not for actual concurrent access. Same underlying cause
as issue #5.

SOLUTION:

This is resolved by the same fix as issue #5. If the Controller owns itself by value (no Arc),
app_event_rx can be a plain AppEventReceiver field with no Mutex wrapper:

    pub struct Controller {
        recorder: Recorder,
        transcriber: Transcriber,
        app_event_rx: AppEventReceiver,
        agent: Agent,
        app_state: AppStateHandle,
        ui: Ui,
    }

The run() method takes self by value and can move app_event_rx into the loop directly.
Shutdown signaling is handled by the separate ShutdownHandle (see issue #5).
